# OpenCam Atlas

OpenCam Atlas is a mock-first MVP for browsing public CCTV and traffic cameras from official sources. It ships with a polished mock catalog, typed provider adapters, server-side API routes, and a responsive dashboard that is ready for live provider integrations later.

## Stack

- Next.js App Router
- TypeScript
- Tailwind CSS v4
- Leaflet via `react-leaflet`
- Local mock provider plus typed provider adapter stubs

## Getting started

1. Install dependencies:

```bash
npm install
```

2. Start the development server:

```bash
npm run dev
```

3. Open [http://localhost:3000](http://localhost:3000)

## Environment

Copy `.env.example` to `.env.local` and adjust values if needed.

```bash
cp .env.example .env.local
```

### Mock mode

Mock mode is the default.

- `ENABLE_REAL_PROVIDERS=false`
- Real provider flags stay off
- All API routes return the normalized mock camera catalog

### Real-provider preparation

The app is structured so live adapters can be added without changing the dashboard or API contracts.

- `lib/providers/types.ts` defines the shared `Camera` model
- `lib/providers/index.ts` decides which providers are active
- `lib/cameras.ts` merges and caches provider output
- Each provider adapter returns normalized `Camera[]`

### Implemented provider adapters

- `mock`: seeded development data
- `hong-kong`: official DATA.GOV.HK traffic snapshot images
- `finland`: official Digitraffic weather cameras
- `singapore`: official data.gov.sg traffic images
- `ohgo`: official Ohio OHGO cameras, requires API key
- `windy`: official Windy Webcams API, requires API key

To turn on live providers:

```bash
ENABLE_REAL_PROVIDERS=true
HONG_KONG_PROVIDER_ENABLED=true
FINLAND_PROVIDER_ENABLED=true
SINGAPORE_PROVIDER_ENABLED=true
```

Optional keyed providers:

```bash
OHGO_PROVIDER_ENABLED=true
OHGO_API_KEY=your-ohgo-key
WINDY_PROVIDER_ENABLED=true
WINDY_API_KEY=your-windy-key
```

## API routes

- `GET /api/cameras`
- `GET /api/cameras/[id]`
- `GET /api/providers`
- `GET /api/health`

There is also a local mock preview route used by the seeded dataset:

- `GET /api/mock-image/[id]`

## Architecture

### Shared model

All providers map into a single normalized shape:

- `id`
- `name`
- `country`
- `city`
- `category`
- `provider`
- `latitude`
- `longitude`
- `previewImageUrl`
- `streamUrl`
- `officialSourceUrl`
- `lastUpdated`
- `refreshSeconds`
- `tags`
- `status`
- `licenseNote`
- `description`

### Provider adapters

Current provider files:

- `lib/providers/mockProvider.ts`
- `lib/providers/hongKongProvider.ts`
- `lib/providers/finlandProvider.ts`
- `lib/providers/singaporeProvider.ts`
- `lib/providers/ohgoProvider.ts`
- `lib/providers/windyProvider.ts`

The first three live public providers are implemented. OHGO and Windy are also implemented, but are disabled until their API keys are configured.

### UI composition

- `app/page.tsx`: homepage data loading
- `components/dashboard-shell.tsx`: search, filters, query-string syncing, favorites, grid/map toggle
- `components/map/camera-map-client.tsx`: Leaflet rendering
- `app/camera/[id]/page.tsx`: detail route
- `components/camera-detail-client.tsx`: detail UI and auto-refresh toggle

## How to add a new provider

1. Create a new file in `lib/providers`, for example `spainProvider.ts`
2. Export a typed fetch function returning `Promise<Camera[]>`
3. Normalize every upstream field into the shared `Camera` interface
4. Add provider metadata and export a `CameraProvider`
5. Register it in `lib/providers/index.ts`
6. Add any required environment flags to `.env.example`
7. Document licensing, rate limits, attribution, and refresh behavior

## What is mocked vs real

### Mocked now

- The 25-camera catalog in `mockProvider`
- Preview images generated by `/api/mock-image/[id]`
- Mock stream pages under `/mock-stream/[id]`
- Trending and popular-tag behavior

### Real now

- Next.js app structure
- Shared provider architecture
- Internal API routes
- Responsive homepage and detail page
- URL-synced filters
- Local favorites via `localStorage`
- Auto-refresh toggle on detail pages
- Official Hong Kong, Finland, and Singapore provider integrations
- Key-gated Windy and OHGO adapters

## Remaining work for live providers

## Notes on compliance

This MVP is intentionally mock-first. The UI repeatedly surfaces source provenance and warns that only official public APIs or explicitly permitted public sources should be connected.
